# MarkDown 문서 생성하기

## MarkDown 문서 형식의 제안

### 📜 마크다운의 역사

**마크다운 (Markdown)** 은 간단하고 읽기 쉬운 텍스트 형식으로 문서를 작성할 수 있도록 설계된 경량 마크업 언어입니다.

#### 창시자

| 정보 | 내용 |
|------|------|
| **창시자** | John Gruber (존 그루버) |
| **협력자** | Aaron Swartz (아론 스워츠) |
| **개발 연도** | 2004년 |
| **발표 장소** | John Gruber의 블로그 (Daring Fireball) |
| **발표 날짜** | 2004년 12월 |

#### 개발 배경

마크다운은 다음과 같은 목표로 개발되었습니다:

- 📝 **가독성**: 마크다운 형식의 텍스트는 다른 형식으로 변환하지 않아도 읽기 쉬워야 함
- 🎯 **간결성**: HTML의 복잡한 태그 없이 간단한 기호만 사용
- 🔄 **호환성**: 웹 표준(HTML)으로 쉽게 변환 가능
- 💻 **실용성**: 이메일, 블로그, 문서 작성에 모두 활용 가능

#### 마크다운의 영향

| 연도 | 사건 | 설명 |
|------|------|------|
| 2004년 | 마크다운 개발 | John Gruber가 공식 발표 |
| 2008년 | GitHub 창립 | 마크다운을 주요 문서 형식으로 채택 |
| 2012년 | GFM 등장 | GitHub Flavored Markdown 출시 |
| 2016년 | 광범위한 채택 | Stack Overflow, Reddit, Slack 등에서 사용 |
| 현재 | 표준화 진행 | CommonMark 등 표준화 노력 진행 중 |

#### 주요 특징

> 💡 **마크다운의 철학**
>
> "A Markdown-formatted document should be publishable as-is, as plain text, 
> without looking like it's been marked up with tags or formatting instructions."
>
> — John Gruber

마크다운의 핵심은 **원본 텍스트도 읽기 쉬워야 한다**는 철학입니다.

## MarkDown 주요 문법

### 1. 제목 (Headings)

마크다운에서는 `#` 기호를 사용하여 제목을 표현합니다. `#`의 개수에 따라 제목의 크기가 결정됩니다.

**문법:**
```markdown
# 제목 1 (가장 큰 제목)
## 제목 2
### 제목 3
#### 제목 4
##### 제목 5
###### 제목 6 (가장 작은 제목)
```

**렌더링 결과:**
# 제목 1
## 제목 2
### 제목 3

**활용 팁:**
- 각 문서는 하나의 `#`으로 시작하는 제목을 가져야 함
- 계층적 구조를 명확히 하기 위해 순차적으로 사용
- 제목 앞뒤로 공백이 필요함

### 2. 텍스트 강조 (Text Emphasis)

텍스트를 굵게, 기울임, 또는 두 가지를 조합하여 표현할 수 있습니다.

**문법:**
```markdown
**굵은 텍스트** 또는 __굵은 텍스트__
*기울임 텍스트* 또는 _기울임 텍스트_
***굵고 기울인 텍스트***
~~삭제선이 있는 텍스트~~
```

**렌더링 결과:**

| 문법 | 결과 | 설명 |
|------|------|------|
| `**텍스트**` | **텍스트** | 굵은 글씨 |
| `*텍스트*` | *텍스트* | 기울임 |
| `***텍스트***` | ***텍스트*** | 굵고 기울임 |
| `~~텍스트~~` | ~~텍스트~~ | 삭제선 |

**활용 예:**
- **중요한 내용**을 강조할 때
- *참고 사항*을 표시할 때
- ~~잘못된 정보~~를 수정할 때

### 3. 리스트 (Lists)

#### 순서 없는 리스트 (Unordered List)

`-`, `*`, `+` 기호를 사용합니다.

**문법:**
```markdown
- 항목 1
- 항목 2
  - 하위 항목 2-1
  - 하위 항목 2-2
- 항목 3
```

**렌더링 결과:**
- 항목 1
- 항목 2
  - 하위 항목 2-1
  - 하위 항목 2-2
- 항목 3

#### 순서 있는 리스트 (Ordered List)

숫자 다음에 점(`.`)을 사용합니다.

**문법:**
```markdown
1. 첫 번째 항목
2. 두 번째 항목
   1. 하위 항목
   2. 또 다른 하위 항목
3. 세 번째 항목
```

**렌더링 결과:**
1. 첫 번째 항목
2. 두 번째 항목
   1. 하위 항목
3. 세 번째 항목

**주의사항:**
- 들여쓰기는 2~4개의 공백 또는 탭 사용
- 숫자는 순서대로 할 필요 없음 (마크다운이 자동 정렬)

### 4. 링크 (Links)

#### 인라인 링크

**문법:**
```markdown
[링크 텍스트](URL)
[링크 텍스트](URL "제목")
```

**예제:**
```markdown
[GitHub](https://github.com)
[구글](https://google.com "구글로 이동")
```

**렌더링 결과:**
- [GitHub](https://github.com)
- [구글](https://google.com)

#### 참조 링크

**문법:**
```markdown
[링크 텍스트][링크 ID]

[링크 ID]: URL
```

**예제:**
```markdown
이것은 [GitHub][1] 링크입니다.

[1]: https://github.com
```

### 5. 이미지 (Images)

**문법:**
```markdown
![이미지 설명](이미지경로)
![이미지 설명](이미지경로 "이미지 제목")
```

**예제:**
```markdown
![시간표](schedule.png)
![GitHub 로고](https://github.com/fluidicon.png "GitHub Logo")
```

**주의사항:**
- 이미지 설명은 이미지가 로드되지 않을 때 표시됨
- 상대 경로 또는 절대 경로 모두 사용 가능

### 6. 코드 (Code)

#### 인라인 코드 (Inline Code)

백틱(backtick) `` ` ``을 사용합니다.

**문법:**
```markdown
이것은 `코드`입니다.
변수 `x = 5`를 선언합니다.
```

**렌더링 결과:**
- 이것은 `코드`입니다.
- 변수 `x = 5`를 선언합니다.

#### 코드 블록 (Code Block)

3개 이상의 백틱(```)으로 감싸고, 언어를 지정할 수 있습니다.

**문법:**
````markdown
```javascript
function hello() {
  console.log("Hello, World!");
}
```
````

**지원 언어:**
```
javascript, python, html, css, java, c, cpp, c#, php, ruby, go, rust, sql, bash, json, yaml, xml, markdown 등
```

**예제:**
```python
def greet(name):
    return f"Hello, {name}!"

print(greet("World"))
```

### 7. 인용문 (Blockquotes)

`>` 기호를 사용합니다.

**문법:**
```markdown
> 이것은 인용문입니다.
> 여러 줄로 작성할 수 있습니다.

> 첫 번째 인용문
>> 중첩된 인용문
>>> 더 깊은 인용문
```

**렌더링 결과:**
> 이것은 인용문입니다.
> 여러 줄로 작성할 수 있습니다.

**활용:**
- 중요한 메시지 강조
- 주의사항 표시
- 명언이나 참고 자료 인용

### 8. 표 (Tables)

`|` 기호와 `-`로 표를 만듭니다.

**문법:**
```markdown
| 헤더 1 | 헤더 2 | 헤더 3 |
|--------|--------|--------|
| 셀 1-1 | 셀 1-2 | 셀 1-3 |
| 셀 2-1 | 셀 2-2 | 셀 2-3 |
```

**렌더링 결과:**

| 헤더 1 | 헤더 2 | 헤더 3 |
|--------|--------|--------|
| 셀 1-1 | 셀 1-2 | 셀 1-3 |
| 셀 2-1 | 셀 2-2 | 셀 2-3 |

#### 텍스트 정렬

**문법:**
```markdown
| 왼쪽 정렬 | 중앙 정렬 | 오른쪽 정렬 |
|:---------|:-------:|----------:|
| 좌측 | 중앙 | 우측 |
```

**렌더링 결과:**

| 왼쪽 정렬 | 중앙 정렬 | 오른쪽 정렬 |
|:---------|:-------:|----------:|
| 좌측 | 중앙 | 우측 |

**정렬 방법:**
- `:---` = 왼쪽 정렬
- `:---:` = 중앙 정렬
- `---:` = 오른쪽 정렬

### 9. 수평선 (Horizontal Rules)

3개 이상의 `*`, `-`, 또는 `_`를 사용합니다.

**문법:**
```markdown
---
***
___
```

**렌더링 결과:**

---

**활용:**
- 섹션 구분
- 문서의 논리적 구분

### 10. 특수 문자 (Escape Characters)

마크다운 문법에 사용되는 문자를 일반 텍스트로 표시하려면 백슬래시(`\`)를 사용합니다.

**문법:**
```markdown
\*별표\*
\[대괄호\]
\#해시태그\#
```

**렌더링 결과:**
- \*별표\*
- \[대괄호\]
- \#해시태그\#

### 마크다운 문법 요약표

| 문법 | 설명 | 예시 |
|------|------|------|
| `# 제목` | 제목 | # 제목 1 |
| `**굵게**` | 텍스트 강조 | **굵은 텍스트** |
| `*기울임*` | 기울임 | *기울임 텍스트* |
| `- 항목` | 리스트 | - 항목 1 |
| `[텍스트](URL)` | 링크 | [GitHub](https://github.com) |
| `` `코드` `` | 인라인 코드 | `const x = 5;` |
| ```` ```코드``` ```` | 코드 블록 | 프로그래밍 코드 |
| `> 인용문` | 인용 | > 이것은 인용 |
| `\|표\|` | 표 | 데이터 표시 |
| `---` | 수평선 | 섹션 구분 |

## GFM 사용과 샘플

GitHub Flavored Markdown (GFM)은 GitHub에서 사용하는 마크다운 방언입니다. 표준 마크다운에 여러 유용한 기능을 추가했습니다.

### 1. 체크리스트 (Task Lists)

체크박스를 사용하여 할 일 목록을 만들 수 있습니다.

```markdown
- [x] 프로젝트 계획
- [x] 개발 환경 설정
- [ ] 기능 구현
- [ ] 테스트 작성
- [ ] 배포
```

**렌더링 결과:**
- [x] 학기 수강신청
- [x] 개강 준비
- [ ] 중간고사 공부
- [ ] 팀 프로젝트 완료
- [ ] 기말고사 대비

### 2. 이모지 (Emoji)

마크다운에 이모지를 추가할 수 있습니다.

| 이모지 | 의미 | 사용 예 |
|--------|------|--------|
| ✅ | 완료 | [x] 과제 제출 완료 |
| ⏳ | 진행 중 | 대기 중인 리뷰 |
| ❌ | 실패/거부 | 요청 거부됨 |
| ⚠️ | 경고 | 주의사항 |
| 🔧 | 수정/개선 | 버그 수정 |
| 📚 | 문서 | 문서 작성 |
| 🎉 | 축하 | 출시 완료! |
| 🐛 | 버그 | 버그 리포트 |

### 3. 코드 강조 (Syntax Highlighting)

프로그래밍 언어를 지정하여 코드 강조 기능을 사용할 수 있습니다.

**JavaScript 예제:**
```javascript
// 시간표 조회 함수
function getSchedule(day) {
  const schedule = {
    "월요일": ["웹 프로그래밍", "소프트웨어공학", "객체지향프로그래밍"],
    "수요일": ["웹 프로그래밍", "소프트웨어공학", "객체지향프로그래밍"],
    "금요일": ["웹 프로그래밍", "소프트웨어공학", "객체지향프로그래밍"]
  };
  return schedule[day] || [];
}

console.log(getSchedule("월요일"));
```

**Python 예제:**
```python
# 학점 계산기
def calculate_grade(score):
    if score >= 90:
        return "A"
    elif score >= 80:
        return "B"
    elif score >= 70:
        return "C"
    else:
        return "F"

print(calculate_grade(85))  # "B"
```

**HTML 예제:**
```html
<!DOCTYPE html>
<html>
<head>
    <title>시간표</title>
</head>
<body>
    <h1>2026학년도 1학기 시간표</h1>
    <p>학번: 20210596</p>
</body>
</html>
```

### 4. 취소선 (Strikethrough)

`~~텍스트~~` 형식으로 취소선을 표현합니다.

```markdown
~~이번주 수요일 강의는 없습니다~~ → 특별 강의가 있습니다.
~~월요일 09:00 강의~~ → 온라인 강의로 변경됩니다.
```

### 5. 자동 링크와 mention

GitHub에서는 URL과 @mention을 자동으로 링크로 변환합니다.

```markdown
학외 정보 확인: https://university.ac.kr
담당 교수 연락: @prof-kim
문의사항: #123 참고
```
